# analyze-zig baseline -- triaged static-analyzer findings (allowlist).
#
# Each line is one accepted finding, normalized to
#   <file>: warning: <message> [<checker>]
# with line:col stripped so edits above a finding don't churn the
# baseline. The check-analyze-zig gate fails on any finding NOT
# listed here -- i.e. a genuinely new analyzer hit.
#
# Triage (2026-06-06): all entries are accepted false positives or
# benign:
#   core.NullDereference -- clang cross-TU FPs: the analyzer cannot
#     see invariants enforced in another TU (arg-count checks,
#     non-NULL guarantees from the caller). mino's prim arity layer
#     guarantees the argv shape these sites assume.
#   deadcode.DeadStores -- defensive initialisations and
#     loop-carried / parse-fallback resets kept for clarity.
#   unix.Malloc (main.c) -- 'end' aliases 'buf' (freed); 'hist_buf'
#     is freed at the same cleanup label. Both FPs.
#   unix.Errno -- errno-clobber after a logged failure on a path
#     that does not consult errno. Benign.
#   unix.Malloc size-zero (emit.c) -- code_len >= 1 for any
#     JIT-eligible bc (a body always ends in OP_RETURN).
#   unix.Malloc size-zero (image.c) -- buf_len comes from ftell() of
#     the image file just written; both the NULL and zero-byte
#     returns from malloc are handled (NULL skips the CRC read loop,
#     leaving crc=0; a non-NULL zero-byte buffer reads zero bytes).
#   unix.Malloc leak (image.c, image_load.c) -- the idt / img_reader
#     arrays are freed at the cleanup labels (img_idt_free /
#     img_reader_free). The analyzer models gc_oom_throw (called
#     transitively through mino_eval_string_ex and mino_print_to) as
#     a normal return, so it sees a path where cleanup is skipped; in
#     fact gc_oom_throw either longjmps to a try frame or aborts, and
#     the embedder's state teardown reclaims anything left. Same
#     false-positive class as the existing mino OOM-throw sites.
#   core.NullDereference (map_owned.c) and deadcode.DeadStores
#     (read_numeric.c) -- baseline drift: read.c was split and
#     map_owned.c is newly visible to the analyzer after the
#     image-serializer TU was added. Same false positives as the
#     existing entries for sibling files.
#
# Regenerate after an intentional change with:
#   ./mino task gen-analyze-baseline
